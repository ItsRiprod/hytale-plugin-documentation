## Texture Organization

### Block Textures Directory

The `BlockTextures/` directory contains ~645 PNG texture files for standard blocks. Textures are organized by naming convention rather than subdirectories.

### Naming Conventions

#### Base Textures

```
{Category}_{Material}.png
```

Examples:
- `Soil_Dirt.png` - Basic dirt texture
- `Rock_Basalt.png` - Basalt rock texture
- `Clay_Blue.png` - Blue clay texture
- `Chalk.png` - Chalk texture

#### Directional Variants

Blocks with different appearances on different faces use suffixes:

| Suffix | Face | Description |
|--------|------|-------------|
| `_Top` | +Y | Top face of block |
| `_Side` | ±X, ±Z | Side faces |
| `_Bottom` | -Y | Bottom face |
| `_Front` | -Z | Front face (oriented blocks) |
| `_Back` | +Z | Back face |

Examples:
- `Soil_Grass_Top.png` - Grass top face
- `Soil_Grass_Side.png` - Grass side face (dirt with grass edge)
- `Bone_Top.png` - Bone block top
- `Bone_Side.png` - Bone block sides

#### Variant Textures

Multiple visual variants use numeric suffixes:

```
{Name}_{Variant}.png
```

Examples:
- `Peachstone_Brick.png`
- `Peachstone_Brick02.png`
- `Peachstone_Brick03.png`
- `Peachstone_Brick04.png`

#### Decorative Variants

| Suffix | Description |
|--------|-------------|
| `_Brick` | Brick pattern |
| `_Cobble` | Cobblestone pattern |
| `_Smooth` | Smooth surface |
| `_Ornate` | Decorative pattern |
| `_Decorative` | Complex decorative pattern |

Examples:
- `Calcite.png` - Base calcite
- `Calcite_Brick.png` - Calcite bricks
- `Calcite_Brick_Smooth.png` - Smooth calcite bricks
- `Calcite_Brick_Ornate.png` - Ornate calcite bricks
- `Calcite_Cobble.png` - Calcite cobblestone

### Tint Gradient Textures

Suffix `_GS` indicates a grayscale texture that uses tint gradients for biome-based coloring:

- `Soil_Grass_Top_GS.png` - Grayscale grass for tinting
- `Soil_Grass_Side_GS.png` - Grayscale grass side
- `Moss_Block_GS_Top.png` - Grayscale moss

## Texture Requirements

### Dimensions

| Requirement | Value |
|-------------|-------|
| Size | Multiples of 32px (32, 64, 96, 128...) |
| Standard block | 32x32 pixels |
| High-detail | 64x64 or 128x128 pixels |
| Format | PNG with alpha channel |

### Alpha Channel

- **Opaque blocks**: No alpha needed
- **Transparent blocks** (glass, leaves): Alpha for transparency
- **Connected blocks**: Alpha for edge blending

### Tint Gradients

For biome-tinted textures (grass, leaves):

1. Create grayscale base texture (`_GS` suffix)
2. Game applies tint gradient based on biome
3. Allows same texture to appear different colors in different regions

## Creating Custom Textures

### Basic Block Texture

```
MyPlugin_Block_Custom.png
```

- Size: 32x32 pixels
- PNG format
- Place in plugin's assets folder

### Directional Block

For blocks with different top/side/bottom:

```
MyPlugin_Block_Custom.png        # All faces (fallback)
MyPlugin_Block_Custom_Top.png    # Top face
MyPlugin_Block_Custom_Side.png   # Side faces
MyPlugin_Block_Custom_Bottom.png # Bottom face
```

### Tinted Block

1. Create grayscale texture:
```
MyPlugin_Grass_Custom_GS.png
```

2. Reference tint gradient in block definition

### Texture Variants

For random variation:
```
MyPlugin_Block_Stone.png
MyPlugin_Block_Stone02.png
MyPlugin_Block_Stone03.png
```

## Color Categories

### Natural Colors by Biome

| Biome | Grass Tint | Soil Color |
|-------|------------|------------|
| Plains | Bright green | Brown |
| Desert | Yellow-green | Tan |
| Cold/Taiga | Blue-green | Gray-brown |
| Jungle | Deep green | Dark brown |
| Volcanic | Orange | Black/gray |
| Burnt | Brown | Charred |

### Clay Colors

Available clay colors:
- Black, Blue, Cyan, Green, Grey
- Lime, Orange, Pink, Purple
- Red, White, Yellow

Each has both rough (`Clay_*`) and smooth (`Clay_Smooth_*`) variants.

### Crystal Colors

- Blue, Cyan, Green, Pink, Purple, Red

Each crystal color has base and `_Top` variants.

## Best Practices

1. **Follow naming conventions** - Use established prefixes and suffixes
2. **Match existing style** - Study vanilla textures for art direction
3. **Use appropriate resolution** - 32x32 for most blocks
4. **Consider tiling** - Textures should tile seamlessly
5. **Optimize file size** - Use PNG compression
6. **Test in-game** - Verify appearance under different lighting
7. **Provide variants** - Multiple variants reduce repetition
8. **Use grayscale for tinting** - Allows biome adaptation
