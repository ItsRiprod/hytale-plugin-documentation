Sound events define how audio files are played in-game, including volume, pitch randomization, attenuation, and layering. They are the core building blocks of Hytale's audio system.

## Sound Event Schema

### Basic Structure

```json title="SFX_Example.json"
{
  "Layers": [
    {
      "Files": ["Sounds/Path/Sound.ogg"],
      "Volume": 0,
      "RandomSettings": {
        "MinPitch": -2,
        "MaxPitch": 2
      }
    }
  ],
  "Volume": 0,
  "Parent": "SFX_Attn_Moderate"
}
```

### Required Fields

| Field | Type | Description |
|-------|------|-------------|
| `Layers` | `array` | One or more audio layers to play |

### Top-Level Properties

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `Volume` | `float` | `0` | Overall volume adjustment (dB) |
| `Parent` | `string` | - | Parent sound event for inheritance |
| `AudioCategory` | `string` | - | Volume category (e.g., `AudioCat_Weapons`) |
| `MaxInstance` | `int` | - | Maximum simultaneous instances |
| `MaxDistance` | `float` | - | Distance at which sound fades out completely |
| `StartAttenuationDistance` | `float` | - | Distance at which attenuation begins |
| `PreventSoundInterruption` | `bool` | `false` | Prevent interruption by same sound |

### Layer Properties

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `Files` | `string[]` | - | Audio file paths (random selection) |
| `Volume` | `float` | `0` | Layer volume adjustment (dB) |
| `StartDelay` | `float` | `0` | Delay before playing (seconds) |
| `RandomSettings` | `object` | - | Randomization parameters |

### RandomSettings Object

| Field | Type | Description |
|-------|------|-------------|
| `MinPitch` | `float` | Minimum pitch offset (semitones) |
| `MaxPitch` | `float` | Maximum pitch offset (semitones) |
| `MinVolume` | `float` | Minimum volume offset (dB) |

## Attenuation Templates

Base templates define spatial audio behavior:

| Template | Max Distance | Start Attenuation | Use Case |
|----------|--------------|-------------------|----------|
| `SFX_Attn_VeryQuiet` | ~15 blocks | ~1 block | Subtle sounds |
| `SFX_Attn_Quiet` | ~25 blocks | ~1 block | Normal sounds |
| `SFX_Attn_Moderate` | ~35 blocks | ~2 blocks | Combat sounds |
| `SFX_Attn_Loud` | ~45 blocks | ~2 blocks | Loud effects |
| `SFX_Attn_VeryLoud` | ~60 blocks | ~3 blocks | Explosions, roars |

## Creating Custom Sound Events

```json title="MyPlugin_CustomSound.json"
{
  "Layers": [
    {
      "Files": [
        "MyPlugin/Sounds/Effect_01.ogg",
        "MyPlugin/Sounds/Effect_02.ogg",
        "MyPlugin/Sounds/Effect_03.ogg"
      ],
      "Volume": 0,
      "RandomSettings": {
        "MinPitch": -1,
        "MaxPitch": 1,
        "MinVolume": -2
      }
    }
  ],
  "Volume": 0,
  "MaxInstance": 5,
  "PreventSoundInterruption": false,
  "AudioCategory": "AudioCat_Weapons",
  "Parent": "SFX_Attn_Moderate"
}
```

<Aside type="tip">
Use the attenuation templates (`SFX_Attn_*`) as parents to inherit spatial audio settings rather than configuring them manually.
</Aside>

## Best Practices

1. **Use Parent templates** - Inherit from attenuation templates for consistent spatial audio
2. **Provide variants** - Multiple files in `Files` array prevent repetitive sounds
3. **Add pitch randomization** - Small pitch variations (+/- 2 semitones) add naturalness
4. **Set appropriate MaxInstance** - Prevent audio clutter from overlapping sounds
5. **Use AudioCategories** - Allow players to adjust volume by category
